## Installation & Troubleshooting (CRITICAL)

### Prerequisites
* Go 1.18 or higher

### Build from Source
Do not compile specific file targets like `go build main.go`. Modern Go requires strict module management.
```bash
git clone [https://github.com/jakeloai/JLFShellGen/](https://github.com/jakeloai/JLFShellGen/)
cd JLFShellGen
go mod init jlfshellgen
go build -o jlfshellgen .

```

### Known Compiler Roadblocks & Fixes

* **`invalid argument` on WSL (Windows Mount Path `/mnt/c/`)**: When compiling directly inside a Windows-mounted directory via WSL, the underlying NTFS file-locking mechanism breaks Go's I/O cache. **Fix:** Move the repository into the native Linux ext4 system (`~/`) before building.
* **`invalid character U+0024` or `syntax error**`: Go's Raw String Literals (backticks) strictly forbid escape characters (`\`). If you modify the `main.go` payload templates (e.g., Perl CGI or Java), ensure variables are correctly encapsulated in Standard String Literals (`"..."`) with proper double-escapes.

### Cross-Compilation

```bash
# Linux AMD64
GOOS=linux GOARCH=amd64 go build -o jlfshellgen-linux .

# Windows AMD64
GOOS=windows GOARCH=amd64 go build -o jlfshellgen.exe .

```

## Usage

### Basic Examples

```bash
# Reverse Shell (Human readable output)
./jlfshellgen -type reverse -lhost 10.0.0.1 -lport 4444 -encode

# Web Shell Arsenal
./jlfshellgen -type webshell -lang php

# Target Specific OS
./jlfshellgen -type reverse -lhost 10.0.0.1 -lport 4444 -os windows

# Pivoting Tunnel Setup
./jlfshellgen -type pivot -lhost 10.0.0.1 -lport 4444 -target 10.0.1.5 -target-port 80 -pivot 10.0.0.2

# Generate Everything for Offline Kit
./jlfshellgen -type all -lhost 10.0.0.1 -lport 4444 -o all_payloads.txt

```

### Pipeline Integration & Output Formats

JLFShellGen is designed to feed downstream automated fuzzers and obfuscators via clean data structures:

```bash
./jlfshellgen -type reverse -lhost 10.0.0.1 -lport 4444 -format raw
./jlfshellgen -type reverse -lhost 10.0.0.1 -lport 4444 -format json
./jlfshellgen -type reverse -lhost 10.0.0.1 -lport 4444 -format markdown

```

**Pipe to external obfuscator:**

```bash
./jlfshellgen -type reverse -lhost 10.0.0.1 -lport 4444 -format raw -o - | ./payloadobfuscator -f - -m all -o obfuscated.txt

```

## Architecture & Technical Details

### Pure Standard Library

JLFShellGen uses only Go standard library packages (`encoding/json`, `net/url`, `crypto/md5`, `strings`, etc.). No external dependencies. Zero CGO. Single static binary.

### Deduplication & Integrity

All payloads are deduplicated in memory using a `map[string]bool` before output, ensuring pipeline efficiency. When writing to a file, an MD5 hash is automatically calculated for payload integrity verification.

```text
Input Layer (CLI Flags)
    |
    v
Environment Filter (OS, Shell, Language)
    |
    v
Payload Generation Engine (Templates)
    |
    v
Encoding Pipeline (Base64, URL, Hex, Pipeline Construction)
    |
    v
Deduplication Engine
    |
    v
Output Formatter (Plain / Raw / JSON / Markdown)

```
